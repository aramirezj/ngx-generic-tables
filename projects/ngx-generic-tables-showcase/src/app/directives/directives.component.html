Directives (standalone each one)

How to use them:

Import the desired directive in section of imports of the module you want to use

GFConversionDirective:

Is a directive that can be applied to an input type text and will transform the value of that input depends of the parameters given

Example working:

Only number Only uppercase Without spaces

Example of code:


    <mat-form-field appearance="outline">
        <mat-label>Only number</mat-label>
        <input matInput type="text" gfConversion="number">
    </mat-form-field>
    <mat-form-field appearance="outline">
        <mat-label>Only uppercase</mat-label>
        <input matInput type="text" gfConversion="uppercase">
    </mat-form-field>
    <mat-form-field appearance="outline">
        <mat-label>Without spaces</mat-label>
        <input matInput type="text" gfConversion="space">
    </mat-form-field>

Types:

   
    export enum GF_TypeConversion {
        UPPERCASE = 'uppercase',
        LOWERCASE = 'lowercase',
        NUMBER = 'number',
        PERCENTAGE = 'percentage',
        SPACE = 'space',
        UPPERCASESPACE = 'uppercaseSpace',
        LOWERCASESPACE = 'lowercaseSpace',
   }

GFMatErrorMessagesDirective:

Directive of component that can be added to a mat-error and it will display automatically error messages, so with only 1 tag of mat-error you can handle all the errors

Example working:

Field required, minlength 3 characters and email format

Example of code:

    
    form:FormGroup = new FormGroup({
        email: new FormControl(null,[Validators.required,Validators.minLength(3),Validators.email])
    });
    

    
        <form [formGroup]="form">
            <mat-form-field  style="width: 50%" appearance="outline">
                <mat-label>Field required, minlength 3 characters and email format</mat-label>
                <input matInput type="text" formControlName="email">
                <mat-error gf-matErrorMessages></mat-error>
            </mat-form-field>
        </form>
